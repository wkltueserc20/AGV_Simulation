## 緣由 (Why)

目前的 AGV 模擬器中，編輯障礙物 (Obstacles) 與設備站點 (Equipment) 只能在純黑的虛擬畫布上憑空編輯或對照座標。在實際的工業部署中，使用者通常擁有廠房的平面圖檔。
本變更旨在引入**「地圖背景匯入與編輯功能」**，讓使用者能夠上傳本地的廠房圖檔作為畫布背景，並能手動定義該圖檔在現實世界中的**「真實物理尺寸 (長與寬，以公尺為單位)」**，進而在此背景圖之上精準地編輯、擺放及對齊實體障礙物與站點，極大提升地圖編輯效率與精準度。

## 變更內容 (What Changes)

- **控制面板元件**: 在前端左側邊欄新增一個「背景地圖配置」控制面板，支援圖片上傳、顯示開關、透明度滑桿、真實物理寬度/高度輸入框（支援等比例鎖定）、偏移量 X/Y 以及旋轉角度的調整。
- **本地狀態與持久化**: 使用 `localStorage` 儲存圖片的 Base64 數據及所有的定位、尺寸、旋轉與透明度參數。重新整理頁面時背景地圖與對齊參數將自動回復。
- **靜態層畫布渲染**: 在 `SimulatorCanvas.tsx` 的靜態快取層 (`updateStaticLayer`) 繪製網格之前，依照使用者設定的比例、偏移與旋轉，以 Canvas Transforms 繪製該背景圖片。

## 能力模組 (Capabilities)

### 新增能力 (New Capabilities)
- `simulator-background-map`: 支援匯入外部圖檔、設定真實物理尺寸（公尺單位）、等比例鎖定、X/Y 偏移、旋轉對齊，並在 Canvas 靜態緩存層進行高流暢度渲染。

### 修改能力 (Modified Capabilities)
- `map-editor`: 擴展原有的地圖編輯功能，使其支援基於背景地圖的障礙物與站點對齊編輯。

## 影響範圍 (Impact)

- `frontend/src/App.tsx`: 整合背景地圖配置的 UI 狀態、控制面板、持久化邏輯，並將參數傳遞給 Canvas。
- `frontend/src/SimulatorCanvas.tsx`: 新增圖片載入與快取邏輯，並在 `updateStaticLayer` 中繪製地圖背景。
- `frontend/src/App.css` (或內聯樣式): 新增美觀的、具備磨砂玻璃质感的控制面板樣式。
